> **Changed in Preview (v0.225).** See [release notes](/releases#0.225).

When using OpenRouter as your assistant provider, you must explicitly select a model in your settings. OpenRouter no longer provides a default model selection.

Configure your preferred OpenRouter model in `settings.json`:

```json
{
  "assistant": {
    "version": "2",
    "provider": {
      "name": "openrouter",
      "default_model": "openrouter/auto"
    }
  }
}
```

The `openrouter/auto` model automatically routes your requests to the most appropriate available model. You can also specify any model available through OpenRouter's API.

### Notes for Reviewer
This is a breaking behavior change for OpenRouter users. Previously, the assistant would fall back to a default fast model (`openrouter/auto`), but now users must explicitly configure a model or OpenRouter won't work. The documentation should make this clear and provide the migration path (setting `openrouter/auto` explicitly if they want the previous behavior).

The change affects the `default_fast_model()` method returning `None` instead of the auto-router model, which means the assistant panel will not have an automatic fallback when OpenRouter is selected without explicit model configuration.
